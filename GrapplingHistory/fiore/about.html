About This Project

How this manuscript reader was built, what tools were used, and where the source material comes from.

What This Is

An interactive reader for the Fior di Battaglia ("Flower of Battle") by Fiore dei Liberi, c. 1410 — a medieval Italian combat treatise covering grappling, dagger, sword, poleaxe, spear, and mounted combat. The primary source is the Getty MS Ludwig XV 13, the most complete of four surviving manuscripts.

This reader was built as the proof-of-concept node for a History of Grappling interactive timeline, part of the GrapplingPrimitives project. It is intended to demonstrate a repeatable pipeline: primary source images → transcription → multi-source translation comparison → interactive reader with lexicon and analysis.

How It Was Built

This project was built almost entirely in a single session using Claude Code (Anthropic's CLI agent, running Claude Opus), directed by a human collaborator. The human provided the project brief, editorial judgement, source material references, domain expertise in grappling history, and critical review of all AI-generated content — including catching fabricated quotes and false attributions.

Pipeline

Getty IIIF API
Fetched the IIIF v2 manifest for MS Ludwig XV 13, downloaded all 94 canvas images at full resolution (3117×4485 px). Script: src/scraper/getty_iiif.py
Claude Vision OCR
8 parallel Claude agents read the manuscript images directly and produced per-folio transcriptions (Middle Italian) + translations (English). 92 files in ~7 minutes. QA showed this is unreliable for the illustration captions (guard names wrong, technique instructions garbled) but usable for the dense prose prologue pages.
Wiktenauer Scrape
Fetched the raw MediaWiki markup from the Fiore de'i Liberi author page (460 KB wikitext, 1.7 MB rendered HTML). Parsed the 8-column concordance table to extract Hatcher and Chidester English translations aligned with Getty folio references. Script: src/scraper/parse_wiktenauer.py
PDF Extraction
Downloaded the Wiktenauer 2016 concordance PDF (15 MB, 532 pages) from Hroarr.com. Used pdftotext for text extraction. Used as a QA reference to verify our Wiktenauer parse against the canonical print compilation.
PD Italian Extract
Extracted the Pisani Dossi (1409) Italian text from the rendered Wiktenauer HTML — Novati's 1902 facsimile transcription, re-transcribed by Michael Chidester. Mapped PD folio references to approximate Getty equivalents. Script: src/scraper/extract_pd_italian.py
WebP Conversion
Converted 94 full-res JPEGs (341 MB) to 1200px-wide WebP (17 MB) using cwebp (libwebp). Originals retained locally, web-optimized versions in the repository.
Static Page
Single index.html file (68 KB) with inline CSS/JS, no build step, no framework. Loads data.json (536 KB) at runtime. Typography follows mise-en-page principles: Spectral serif for body, Inter for UI, 64ch measure, tiered spacing.

What Claude Did vs. What the Human Did

Claude (via Claude Code CLI):

  • Wrote all scraping, parsing, and build scripts
  • Performed the OCR pass on all 92 folios (8 parallel agents)
  • Parsed the Wiktenauer wikitext and extracted translations
  • Built the HTML/CSS/JS reader page
  • Wrote the initial lexicon entries and sentiment analysis
  • Generated the text analysis statistics
  • Managed the git repository and deployment

The human (Gerald / Jelaludo):

  • Provided the project brief and domain expertise in grappling history
  • Directed the overall architecture and editorial priorities
  • Identified and provided reference translations (Hatcher, Chidester)
  • Caught fabricated quotes — Claude synthesized "prese d'amore" as a Getty term when it only appears in the Pisani Dossi; invented a "three tiers of violence" framework that oversimplified what Fiore actually wrote
  • Caught false attributions — quotes from different folios and manuscripts stitched together to appear as a unified argument
  • Provided the Sbarra watercolor illustration
  • Edited lexicon entries for accuracy via the in-browser edit mode
  • Insisted on the search feature to verify claims against source text
  • Defined the vocabulary mapping (solaço/ira, sbarra, oltrança, etc.) with corrections to ensure no editorial invention was presented as Fiore's words

Accuracy Caveats

  • Claude Vision OCR of the illustration pages is unreliable. Guard names, technique instructions, and tactical logic are frequently garbled. The Hatcher/Chidester scholarly translations should be treated as authoritative; the "Raw OCR" tab is a first-pass reference only.
  • Prologue OCR (fol. 1r, 2r, 4r) is substantially better, having been cross-referenced against existing OCR work. Still not scholarly-grade.
  • Folio-to-section mapping is approximate, based on the Wiktenauer concordance and manuscript structure. Some boundary folios may be miscategorized.
  • Pisani Dossi ↔ Getty alignment is approximate. The two manuscripts have different structures and the PD is a condensed redaction. Mapping is based on concordance table co-occurrence, not scholarly apparatus.
  • The lexicon and analysis sections contain editorial interpretation, clearly marked. Every quoted passage links to its source folio and tab for verification. If a claim cannot be verified by clicking its source link, treat it with skepticism.

Lessons Learned

  • Vision OCR on medieval script is not reliable for short captions. Claude reads the dense prose prologue pages reasonably well but fails on the illustration captions — the 15th-century Venetian cursive hand defeats the model. Scholarly transcriptions remain essential.
  • AI confidently fabricates scholarly synthesis. Claude stitched quotes from different manuscripts and folios into unified arguments, invented the term "prese d'amore" as if it were a Getty concept, and constructed a "three tiers of violence" framework that oversimplified Fiore's actual distinctions. Every claim needs source verification.
  • The search feature was the most important addition. The human insisted on it specifically to verify AI claims against source text. It caught multiple errors that would have shipped otherwise. If you're building a scholarly interface with AI assistance, build the verification tool first.
  • Different manuscripts by the same author use different words for the same concepts. The Getty says solaço where the Pisani Dossi says d'amore. This isn't just a variant spelling — it may reflect evolving thought across the ~decade separating the manuscripts. Multi-manuscript comparison is not optional.
